# Cloud-Neutral 资讯雷达｜交付与测试 · Grafana k6

## Grafana k6

Grafana k6 是 Cloud-Neutral Tool Map 中 **交付与测试（Performance & Quality Gate）** 这一层里，最具工程化特征的性能测试工具之一。它并不试图模拟完整的“真实用户世界”，而是明确聚焦在一个问题上：

> **在可控负载模型下，系统什么时候开始不可靠。**

与传统 GUI 压测工具不同，k6 从设计之初就将性能测试视为一种 **可版本化、可自动化、可回归的工程资产**。

---

## 项目主要特性

- 基于 JavaScript 的负载测试脚本
- CLI 驱动，天然适配 CI/CD
- 面向 HTTP / API / 微服务
- 内置指标与阈值（Threshold）机制
- 与 Grafana / Prometheus 深度集成

k6 的定位不是“压一次出报告”，而是： ****让性能回退成为一种可被自动拦截的交付信号。**

---

## 优缺点

| 优点 | 局限 |
|---|---|
| 测试即代码，易维护 | 非浏览器级测试 |
| 适合自动化与回归 | 用户行为抽象 |
| 云原生 / 容器友好 | 不覆盖前端体验 |
| 指标体系清晰 | 不适合一次性压测 |

---

## 适用场景

| 适合 | 不适合 |
|---|---|
| API / 微服务性能验证 | 页面渲染性能 |
| CI 阶段性能回退检测 | 强 UI 交互流程 |
| 容量评估与极限测试 | 临时人工压测 |
| 与可观测体系联动 | |

---

## 工程判断

Grafana k6 并不是“性能测试的全部”，而是**性能工程化的起点**。它清晰地站在这样一个工程边界上：

> **在持续交付过程中，稳定地回答“这次改动有没有让系统变慢”。**

在现代体系中，一个合理的位置是：

> Playwright 验证前端流程，  
> **k6 验证后端承压能力**，  
> Grafana 统一观察系统是否进入不稳定区。
